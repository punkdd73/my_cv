倉儲物流營運管理

1. 倉儲物流營運管理

負責進出庫、逆物流及庫存盤點之標準作業，確保庫存帳務一致，管控庫存損耗及費用成本。

管理與第三方物流之合作關係，日常運作監控，服務品質稽核及異常處理。

適應高變動的彈性營運規劃

2. 適應高變動的彈性營運規劃

設計具彈性的人力與流程方案，以動態排程配合電商檔期或突發需求，確保量能穩定。

透過數據監控持續分析營運盲點，並提出可執行的改善計畫。

自動化與否皆可

3. 自動化與否皆可

熟悉ERP邏輯及系統架構，純人工、半自動或高度自動化流程皆可勝任。

具Vibe Coding能力，結合AI開發微型工具，將局部低效人工作業轉為自動化流程。

跨部門的供應鏈協調中心

4. 跨部門的供應鏈協調中心

擔任物流窗口代表，協同採購、行銷、財務、資訊等部門解決進銷存與物流問題。

透過客訴與異常個案處理，推動「跨部門問題溯源」與「整體預防機制」規劃工作。

架構規劃及數據決策

5. 架構規劃及數據決策

針對多平台、多通路、多屬性作業，規劃具擴充性之倉儲物流作業架構及流程。

負責倉儲物流相關數據的整理編輯，提供具參考價值的資訊供決策層追蹤判斷使用。

CORE COMPETENCY

戰略與執行雙向轉譯實例

將出自台北市信義區的抽象目標，無縫轉譯為桃園市楊梅區混亂現場的明確指令，或反向提供具價值的實務建議。

信義區 = 總部 轉譯員 = 我 楊梅區 = 倉庫
信義區

「本年度3PL倉儲物流費用目標降低15%以上。」

要求在不損及出貨服務水準的前提下，大幅削減倉儲固定營運開支。

「全面控管商品效期，避免即期報廢損失。」

要求倉儲單位協助營運端動態掌握有效期限，確保商品去化週期健康。

「大型檔期多端點銷售，確保庫存控管與整體出貨效率。」

要求在預購、團購、一般訂單等多端點銷售情況下，掌握全局庫存動態，避免超賣，並於指定區間完成出貨。

轉譯員

「找出隱藏成本，制定談判籌碼」

交叉比對合約單價與實際帳單，抓出「非標準作業」的溢價點；親自到現場「參與」實際作業，觀察儲位率、進出貨流程，擬定談判策略。

「盤點現狀與制定模型」

了解業務不喜歡自行查閱複雜庫存報表的痛點，改以「極簡化認知」切入；評估現有系統極限，決定不改動底層架構，改以外部輕量工具克服。

「建構檔期戰情雲端表，防範超賣與斷貨」

面對預購/團購多端點爆單，搭建雲端動態庫存控管表，讓業務、行銷、客服、倉庫等多方即時對齊數據，並預先備妥自動化工具(如SCM操作外掛)維持操作穩定。

楊梅區

「拆解計價模組 + 改善作業流程」

與3PL談判合約降價以外，透過協商日常作業優化，例如併板、併箱、包材管控、附加服務等策略，創造額外降幅。

「內部開發自動化預警系統」

以AI開發自動化預警系統，透過Python腳本每日自動掃描，提前30天將即期品推播至工作群組，責任歸屬一目了然，讓業務在報廢前主動掌握去化節奏。

「預調量能待命 + 臨時SOP預加工」

透過歷史數據與現場進行沙盤推演，預先協調人力與預加工，擬定突發事件的應對SOP，確保大促期間0擠爆、0延遲。
